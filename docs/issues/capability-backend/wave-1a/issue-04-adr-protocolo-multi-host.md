# ADR-004: Protocolo multi-host de desenvolvimento e handover interno

**Tipo**: Process  
**Labels sugeridas**: `adr`, `process`, `development`, `multi-host`

## Contexto

`HANDOVER.md` funciona. O problema de ferramenta interna e quando ela comeca a pedir cidadania no produto. Esta ADR serve para evitar esse casamento indevido.

## Objetivo

Fixar que:

1. `HANDOVER.md` e ferramenta interna de oficina
2. nao faz parte do produto
3. serve para coordenacao entre Codex, Claude e Antigravity
4. toda decisao duravel deve subir para GitHub

## Entregaveis

1. ADR-004 escrito e commitado
2. definicao curta dos usos validos do handover
3. definicao do que sempre sobe para issue, ADR ou PR

## Criterio de aceite

1. ninguem mais confunde backstage com feature
2. o processo multi-host fica documentado sem virar burocracia ornamental
