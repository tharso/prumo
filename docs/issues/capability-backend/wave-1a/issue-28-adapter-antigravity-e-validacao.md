# Especificar adapter ideal do Antigravity e protocolo de validacao real no host

**Tipo**: Host adapter / Validation  
**Labels sugeridas**: `host`, `antigravity`, `ux`, `wave-1`

## Contexto

O Antigravity foi o host que mais bateu onde doia: latencia de tool chaining, excesso de wrapper, perda de persona e burocracia de limpeza. Se ele entrar no fim da fila, a chance de descobrir tarde demais e alta demais.

## Objetivo

Transformar Antigravity em trilho formal de validacao de experiencia e em adapter de referencia para host local forte.

## Escopo

1. restricoes e vantagens do host
2. politica de uso de ferramentas nativas vs wrappers do runtime
3. limites aceitaveis de granularidade das capacidades
4. expectativas de persona e disciplina conversacional
5. criterios de validacao de UX no host

## Entregaveis

1. spec do adapter Antigravity
2. protocolo de validacao real no host
3. lista de anti-patterns a evitar

## Criterio de aceite

1. o projeto para de desenhar host local forte no escuro
2. o feedback do Antigravity entra cedo o bastante para mudar a obra, nao so comentar a pintura
