# Definir portabilidade de persona e disciplina conversacional do Prumo

**Tipo**: Product / Host UX  
**Labels sugeridas**: `ux`, `persona`, `hosts`, `product`

## Contexto

Se a alma do produto ficar subentendida, cada host vai soar como um call center de um planeta diferente. O plugin antigo tinha restricoes de ritmo, tom e disciplina que nao brotam espontaneamente do modelo so porque a gente acha bonito.

## Objetivo

Evitar que a personalidade do Prumo evapore quando a UX voltar para o host.

## Escopo

1. tom de voz
2. limites de listagem
3. uma pergunta por vez quando aplicavel
4. disciplina de interacao
5. persona in-prompt / adapter prompt
6. o que e invariavel e o que pode variar por host

## Entregaveis

1. contrato de persona portavel
2. conjunto inicial de regras de disciplina conversacional
3. criterios para adaptacao por host sem descaracterizacao

## Criterio de aceite

1. Claude, Codex e Antigravity deixam de depender de telepatia para soar como Prumo
